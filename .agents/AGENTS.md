# BIOSILIN Project Rules

- **Local Modification & Testing**: Always make and run changes locally first. Do not commit or push to Git automatically.
- **Push Approval**: The user must explicitly review and approve before any changes are committed or pushed to the `Biosilin-DEV` branch.
- **Document Changes**: Document all changes made in a text file (e.g., a log or changelog file) that is committed and pushed to Git alongside the changes.
- **MCP Server Selection**: Automatically select the most appropriate MCP server(s) for the task without asking. Use the minimum required set, combine only when it clearly improves the result, and choose the most reliable/relevant option. Only ask for clarification if no clear choice exists or if credentials/permissions/setup are required.
